# Пошаговое руководство «Начало работы»

## Проблема в Cursor: «Не удалось открыть редактор»

Если в **Cursor** при открытии руководства появляется ошибка «Не удалось открыть редактор из-за непредвиденной ошибки», а в VS Code всё открывается нормально, попробуйте по порядку:

1. **Проверить консоль**  
   `Справка` → `Переключить средства разработчика` → вкладка **Console**. Откройте руководство снова и посмотрите текст ошибки (красные строки). Это поможет понять, падает ли загрузка изображения, парсинг markdown и т.п.

2. **Убедиться, что есть файл-заглушка**  
   В папке `walkthrough/images/` должен быть `placeholder.svg`. В `walkthrough/*.md` везде используется ссылка на него. В одном из файлов раньше была ссылка на `placeholder.png` — если такого файла нет, в Cursor это могло приводить к сбою.

3. **Временно убрать картинки**  
   Если ошибка сохраняется, в каждом `walkthrough/*.md` закомментируйте или удалите строку с изображением:
   - было: `![Место для скриншота](images/placeholder.svg)`
   - станет: пустая строка или `<!-- ![...](images/placeholder.svg) -->`  
   Перезапустите Cursor и откройте руководство. Если после этого оно открывается — проблема, скорее всего, в отображении/загрузке изображений в Cursor.

4. **Обновить Cursor**  
   Установите последнюю версию Cursor — в новых сборках нередко чинят совместимость с walkthrough и webview.

5. **Сообщить в Cursor**  
   Если ничего не помогло, опишите проблему в поддержке или на форуме Cursor: что делаете (открываете пошаговое руководство расширения), что видите (ошибка в редакторе), и приложите скрин/текст из консоли разработчика.

## Структура

- `*.md` — контент правой панели для каждого шага (скриншот + при необходимости свой текст).
- `images/` — скриншоты или заглушка `placeholder.svg`.
- Описания шагов и кнопка «Выполнить» задаются в `package.json` (`contributes.walkthroughs`).
