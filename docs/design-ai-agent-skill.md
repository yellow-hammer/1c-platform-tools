# Предложение по реализации: навык для AI-агентов (Issue #56)

## Цель

Реализовать возможность добавлять **навыки (Agent Skills)** для AI-агентов (Cursor, Claude Code и др.) через интерфейс расширения 1C: Platform tools — по аналогии с созданием `packagedef` из шаблона в панели «Инструменты 1С».

**Главное:** навык не просто кладёт файлы в проект — он **подсказывает агенту, как использовать команды расширения**. Агент должен предлагать пользователю вызывать команды 1C: Platform tools (или вызывать их сам, если среда это поддерживает), а не запускать внешние скрипты. То есть операции 1С (загрузка конфигурации, сборка обработки, выгрузка расширения и т.д.) выполняются **нашими командами расширения** (например `1c-platform-tools.configuration.loadFromSrc`), а текст навыка — это инструкция для агента: какая задача → какая команда.

---

## Назначение навыка: вызов команд расширения

Навык должен **подсказывать агенту, как использовать команды расширения 1C: Platform tools**:

1. **Описание в SKILL.md** — когда нужна операция 1С (загрузить конфигурацию, собрать обработку, выгрузить расширение и т.д.), агент читает навык и узнаёт, какую **команду VS Code** предложить или выполнить.
2. **Таблица «задача → команда»** — в тексте навыка явно указаны идентификаторы команд (`1c-platform-tools.configuration.loadFromSrc`, `1c-platform-tools.externalProcessors.build` и т.д.), чтобы агент мог рекомендовать пользователю выполнить команду из палитры или из панели «Инструменты 1С», либо (если среда позволяет) вызвать команду сам через API.
3. **Итог:** операции 1С выполняются **расширением** (vrunner, env.json, настройки путей), а не bat-файлами. Файлы навыка в проекте — это только инструкции для агента; при необходимости можно добавить bat/скрипты как запасной вариант для сред без расширения.

---

## Варианты реализации

### Вариант 1: Только расширение (шаблоны навыков в проекте)

**Идея:** В расширении хранятся шаблоны навыков. Команда в дереве «Инструменты 1С» копирует шаблон в выбранную папку workspace (например `.cursor/skills/1c-platform-tools`). Содержимое навыка — **инструкции для агента с привязкой к командам расширения** (см. выше), а не набор bat-скриптов.

**Плюсы:**

- Привычный UX: как «Инициализировать проект» / «Только packagedef»
- Одна кодовая база, обновление навыка вместе с расширением
- Агент подхватывает навык и знает, какие команды расширения вызывать
- Операции выполняются расширением (единые настройки, vrunner, env)

**Минусы:**

- Разные IDE могут ожидать навыки в разных путях (`.cursor/skills` vs `.claude/skills`) — решается выбором папки при установке

---

### Вариант 2: Отдельный MCP-сервер

**Идея:** Отдельный проект (например `1c-platform-tools-mcp` или интеграция в другой MCP) предоставляет инструменты (tools): `build_epf`, `dump_epf`, `load_config`, `dump_config` и т.д. Агент вызывает их по Model Context Protocol.

**Плюсы:**

- Единый протокол для любых агентов с поддержкой MCP
- Не привязан к bat-файлам и ОС
- Можно вызывать логику без терминала (если MCP вызывает vrunner/1cv8 сам)

**Минусы:**

- Дублирование/расхождение логики с расширением (vrunner, пути, env)
- Нужен запущенный MCP-процесс и настройка в Cursor/IDE
- Отдельный репозиторий/пакет и поддержка

---

### Вариант 3: Гибрид (рекомендуется)

1. **В расширении:** команда «Добавить навык для AI» в панели «Инструменты 1С»:
   - QuickPick: «Какой навык добавить?» → например «1C: Platform tools — команды для агента»
   - Целевая папка: по умолчанию `.cursor/skills/1c-platform-tools` (или выбор: Cursor / Claude / указать вручную)
   - Копирование из `resources/skills/1c-platform-tools/` в workspace: в первую очередь **SKILL.md** с инструкциями и таблицей «задача → command ID»; при необходимости — дополнительные файлы (например пример env).

2. **MCP — отдельно (по желанию):** позже можно вынести в отдельный проект MCP с tools; расширение упоминает его в документации.

Итог: агент получает в проекте файлы навыка, которые **подсказывают вызывать команды расширения**; сами операции выполняются расширением.

## Детали реализации (Вариант 3, часть «расширение»)

### 1. Структура шаблона в репозитории

```
resources/
  skills/
    1c-platform-tools/
      SKILL.md              # главный файл: описание + таблица «задача → command ID»
      (при необходимости)   # примеры env.json / подсказки в README внутри навыка
```

**Главное — содержимое SKILL.md.** В нём должны быть:

- **Frontmatter** (name, description) для краткого контекста агента (~100 токенов).
- **Инструкция:** для операций с 1С используй команды расширения 1C: Platform tools; предлагай пользователю выполнить команду из палитры (Ctrl+Shift+P) или из панели «Инструменты 1С».
- **Таблица соответствия** «что нужно сделать → какую команду вызвать» с точными идентификаторами команд (command ID).

Пример тела навыка (фрагмент):

```markdown

description: Операции с платформой 1С в этом проекте выполняются командами расширения 1C: Platform tools. Используй команды расширения (палитра или панель «Инструменты 1С»), не запускай внешние скрипты.

# Команды 1C: Platform tools для агента

Выполняй операции 1С через команды расширения. Предложи пользователю выполнить команду из палитры (Ctrl+Shift+P) или из панели «Инструменты 1С».

## Конфигурация

| Задача | Command ID |
|--------|------------|
| Загрузить конфигурацию из src/cf | `1c-platform-tools.configuration.loadFromSrc` |
| Загрузить только изменения (git diff) | `1c-platform-tools.configuration.loadIncrementFromSrc` |
| Выгрузить конфигурацию в src/cf | `1c-platform-tools.configuration.dumpToSrc` |
| Выгрузить изменения в src/cf | `1c-platform-tools.configuration.dumpIncrementToSrc` |
| Собрать 1Cv8.cf из src/cf | `1c-platform-tools.configuration.build` |
| Разобрать 1Cv8.cf в src/cf | `1c-platform-tools.configuration.decompile` |

## Расширения

| Задача | Command ID |
|--------|------------|
| Загрузить расширение из src/cfe | `1c-platform-tools.extensions.loadFromSrc` |
| Выгрузить расширение в src/cfe | `1c-platform-tools.extensions.dumpToSrc` |
| Собрать *.cfe из src/cfe | `1c-platform-tools.extensions.build` |
| Разобрать *.cfe в src/cfe | `1c-platform-tools.extensions.decompile` |

## Внешние обработки и отчёты

| Задача | Command ID |
|--------|------------|
| Собрать внешнюю обработку | `1c-platform-tools.externalProcessors.build` |
| Разобрать внешнюю обработку | `1c-platform-tools.externalProcessors.decompile` |
| Собрать внешний отчёт | `1c-platform-tools.externalReports.build` |
| Разобрать внешний отчёт | `1c-platform-tools.externalReports.decompile` |

## Запуск

| Задача | Command ID |

| Запустить Предприятие | `1c-platform-tools.run.enterprise` |

### 2. Команда и дерево

- **Команда:** например `1c-platform-tools.skills.add` («Добавить навык для AI»).
- **Где показывать:** группа в `TREE_GROUPS` в `treeStructure.ts` — например новая группа «Навыки для AI» с пунктом «Добавить навык для AI» (QuickPick выбора навыка: пока один — «1C: Platform tools (команды для агента)»).
- **Регистрация:** `package.json` → `contributes.commands`; `commandRegistry.ts` → вызов нового класса/метода (например `SkillsCommands.addSkill()`).
- **Условие видимости:** в контексте проекта 1С; можно разрешить добавление навыка и без открытого проекта (тогда выбор папки обязателен).

### 3. Сценарий выполнения команды

1. Пользователь вызывает команду из дерева «Инструменты 1С» или из палитры.
2. QuickPick: «Выберите навык» → варианты (пока один: «1C: Platform tools — команды для агента»).
3. «Куда установить?»: «Для Cursor» → `.cursor/skills/1c-platform-tools`, «Для Claude Code» → `.claude/skills/1c-platform-tools`, «Указать папку…» → `showOpenDialog`. Корень — workspace root.
4. Если папка уже существует и содержит SKILL.md — спросить «Перезаписать?».
5. Копирование файлов из `resources/skills/1c-platform-tools/` в выбранную папку.
6. Сообщение: «Навык установлен. Агент будет использовать инструкции из SKILL.md и предлагать выполнять команды расширения.»

### 4. Технические моменты

- **Класс команд:** `SkillsCommands` в `src/commands/skillsCommands.ts`, по аналогии с `DependenciesCommands` (только копирование файлов, vrunner не нужен).
- **Копирование:** `fs.cp` (Node 16+) рекурсивно из `path.join(extensionPath, 'resources', 'skills', skillId)`.
- **Локализация:** строки на русском в UI.
- **Логирование:** через `logger` из `src/logger.ts`.
